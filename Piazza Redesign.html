Project Overview

As UCSD students, we identified significant usability issues with Piazza's mobile app. While the platform serves as a crucial tool for course discussions and announcements, its mobile interface presented several challenges that hindered the user experience. Our team set out to redesign key components of the mobile app to better serve students' needs.

The Challenge

Improve the mobile experience of Piazza while maintaining its core functionality as a course discussion platform, focusing on making it more intuitive and efficient for students to use on their mobile devices.

Research Process

User Testing: Round I

We conducted preliminary user tests with four college-aged students to understand their experience with the current Piazza mobile app.

Interview Structure

  • Previous experience with Piazza mobile/desktop
  • Primary use cases
  • Task completion assessment
  • Follow-up questions about pain points

Test Tasks

  • Navigate to specific class discussion board
  • Mark multiple posts as 'read'
  • Post a question with an image attachment
  • Filter posts by 'Unread'
  • Find and respond to specific posts

Key Findings

Recognition vs. Recall

Users struggled to remember the location of frequently used features, particularly notification settings and marking posts as read.

Flexibility Issues

Users attempted to use intuitive gestures (like swiping) that weren't supported, leading to frustration.

System-Real World Mismatch

Confusion arose from inconsistent terminology and unintuitive discussion thread interactions.

Testing & Iterations

Our iterative testing process revealed several key insights that guided our final design decisions:

Navigation

Users found the bottom navigation bar more intuitive and accessible compared to hidden menus.

Content Organization

Color-coded categories and improved filtering options helped users find relevant content more quickly.

Interaction Design

Simplified post interaction patterns reduced confusion and improved task completion rates.
